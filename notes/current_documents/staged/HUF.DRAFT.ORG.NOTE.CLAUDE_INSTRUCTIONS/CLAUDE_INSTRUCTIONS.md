# Instructions to Claude: generating HUF-coded "Current Documents"
Goal: Every new document generated by the collective must be **HUF-coded at creation time** so we stop filename collisions.

## Output lane
All new outputs go to:
- `notes/current_documents/inbox/YYYY-MM/` (raw)
Then promoted to:
- `notes/current_documents/staged/<DOC_ID>/` (ID assigned + header applied)

No document enters `docs/` unless it is Markdown and considered release-lane.

## Two-line header (mandatory)
Every document must start with:

Line 1:
HUF-DOC: <DOC_ID> | HUF:1.1.8 | DOC:<DOC_VERSION> | STATUS:<draft/reviewed/release> | LANE:<draft/release/legacy> | RO:<Responsible Operator>

Line 2:
CODES: <comma tags> | ART: CM, AS, TR, EB | EVID:E0-E4 | POSTURE: DEF/OP/INT/THM | WEIGHTS: OP=.. TOOL=.. PEER=.. | CAP: OP_MIN=0.51 TOOL_MAX=0.49 | CANON:<canonical path>

## Doc ID rules (stable identifier)
DOC_ID format:
HUF.<LANE>.<DOMAIN>.<TYPE>.<SLUG>

- LANE: DRAFT / REL / LEGACY
- DOMAIN: ORG / LRN / BOOK / PARTNER / CASE / WIKI / SOFTWARE
- TYPE: PAGE / NOTE / PRIMER / EXHIBIT / TRACE / TEMPLATE / MANUSCRIPT
- SLUG: uppercase underscores (stable)

Example:
HUF.DRAFT.PARTNER.PAGE.RAMSAR_CONTACT_PRIMER

## Status discipline (slow release)
- draft: moving
- reviewed: stable structure + header + manifest entry
- release: safe to circulate; human-only release gate passed
- deprecated: superseded; kept for trail

## Operator control contract (HUF-51/49)
Must declare weights; must satisfy:
- OP >= 0.51
- TOOL <= 0.49
- OP + TOOL + PEER = 1.0
Release decisions remain human-only.

## After generating a batch
Run inventory (manual):
  .\.venv\Scripts\python scripts\doc_inventory.py --root . --write --merge

Then resolve collisions:
- If two files claim same DOC_ID, only one can be canonical; update the other’s DOC_ID or mark deprecated.

Deliverable to ChatGPT:
- zip of the staged folder(s) + updated manifest, so ChatGPT can map items to release destinations.
